# /session-briefing — 세션 시작 브리핑

프로젝트 CLAUDE.md 지침에 따라 **대화형 세션을 열면 가장 먼저** 이 커맨드를 실행한다(사람이 직접 입력하지 않아도).
헤드리스 실행에서는 호출하지 않는다.

## 절차

1. **최근 실행 이력 요약** — `.claude/automation/log.md` 의 최근 항목을 읽고 태그별로 집계한다.
   예: "자동 처리·push 완료 2건, 판단 대기 1건, 직접 수정 1건". `[이슈]` 와 `[직접]` 을 구분해 보여준다.
   (로그 파일이 없으면 "아직 실행 이력 없음"으로 안내.)

2. **미push 커밋 확인 + 배치 마감 재개** — 현재 브랜치에 push 안 된 커밋이 있는지 보여준다:
   `git log --oneline @{u}.. 2>/dev/null` (upstream 이 없으면 `git log --oneline origin/<branch>..` 로 대체).
   push는 **배치**(이슈마다 따로 밀지 않고 모아서 한 번)이므로, 여기 남는 건 보통 **배치 대기분**
   (커밋만 하고 아직 push·close 안 한 이슈)이거나 `[직접]` 즉석 수정, 또는 push 실패분이다.
   - 각 미push 커밋의 메시지에서 `(#N)` 을 뽑아, **그 이슈가 아직 열려 있으면 마감 미완**(커밋만 있고 push/[템플릿3]/close 안 됨)이다.
   - 이런 이슈가 있으면 4단계 질문에서 **"미push 커밋 M건(이슈 #.. 마감 대기) — 지금 모아서 push하고 [템플릿3]+close로 마감할까요?"** 로 함께 제시해 재개할 수 있게 한다(push → 각 이슈 [템플릿3] → close).

3. **판단 대기 항목 목록** — 판단 대기의 **SSOT는 GitHub 열린 이슈**다(헤드리스는 `.claude/` Write 불가라
   `decisions/ISSUE-*.md` 를 못 남긴다 — 로컬 문서만 보면 헤드리스가 처리한 판단 대기를 놓친다). 다음을 종합한다:
   - `gh issue list --state open --json number,title` 로 열린 이슈를 나열하고, 각 이슈에 봇이 남긴
     **[템플릿2]("🔍 검토가 필요합니다")** 코멘트가 있는지 `gh issue view N --json comments` 로 확인한다.
     → **[템플릿2] 있음 = 판단 대기**(옵션 게시 완료, `/resolve-issue` 대상) / **없음 = 미분류**(다음 `/issue-triage` 대상).
   - 보조로 `.claude/automation/log.md` 의 "판단 필요" 항목과 `.claude/automation/decisions/ISSUE-*.md`
     (대화형 세션이 남긴 경우)도 함께 훑어 각 한 줄 요약에 반영한다.
   - **각 판단 대기 이슈마다 [템플릿2] 코멘트에서 검토된 옵션(A/B/C…)과 추천안을 뽑아 함께 제시한다** —
     담당자가 브리핑만 보고 바로 `/resolve-issue N <옵션>` 을 고를 수 있도록 옵션 라벨을 간결히(각 한 줄) 보여준다.
   - (열린 이슈가 없으면 "판단 대기 항목 없음"으로 안내한다.)

4. **질문** — 위를 종합해 "판단 대기 항목 중 지금 진행할 것이 있나요? (예: `/resolve-issue 15 B`)
   아니면 새 이슈를 확인할까요? (`/issue-triage`)" 라고 사용자에게 묻는다.

## 출력 형식(예시)

```
📋 세션 브리핑 (<날짜/시각>)
• 최근 자동 처리: [이슈] 2건 완료·push, 판단 대기 1건 / [직접] 1건
• 미push 커밋: (없음)
• 판단 대기(GitHub 열린 이슈, [템플릿2] 게시됨):
   - #15 발표 스크립트 저장 방식 결정 대기
       A) 로컬 저장(단순, 오프라인) / B) 서버 저장(동기화, 추천) / C) 둘 다(복잡)
       → 추천: B
진행할 항목을 알려주세요 (`/resolve-issue 15 B` 처럼 옵션 선택, 또는 `/issue-triage` 로 새 이슈 확인).
```
