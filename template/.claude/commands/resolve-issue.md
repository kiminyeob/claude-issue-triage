# /resolve-issue — 판단 대기 이슈 처리

인자: `$ARGUMENTS` = `<이슈번호> <옵션문자>` (예: `15 B`). `$1`=이슈번호, `$2`=옵션.

## 절차

1. **옵션 복원** — `.claude/automation/decisions/ISSUE-$1.md` 를 읽는다.
   문서가 없으면 GitHub 이슈에 남아 있는 **[템플릿 2]** 코멘트를 `gh issue view $1 --comments` 로 읽어 옵션 내용을 복원한다.
   지정된 옵션 `$2` 의 구현 방향과 트레이드오프를 확인한다. (옵션이 불명확하면 사용자에게 되묻는다.)

2. **구현** — 해당 옵션대로 코드를 수정한다.
   [`issue-triage.md`](issue-triage.md) 상단 ✏️ 「프로젝트 고유 원칙」이 있으면 준수하고,
   근거 없는 제품/평가 기준을 새로 만들지 않는다(필요 시 사용자 확인).

3. **검증** — 성격에 맞게 ✏️ 「검증 명령」(issue-triage.md 상단)을 돌려 통과 확인.

4. **커밋(push는 보류·배치)** — `git add <바뀐 파일들>` → `git commit -m "<type>: <요약> (#$1)"` (한 이슈 = 한 커밋).
   **여기서 push하지 않는다.** 여러 이슈를 이어서 처리할 수 있으므로 커밋만 쌓아두고 push는 배치로 한 번에 민다
   (커밋 해시가 이슈를 구분하므로 이슈마다 따로 push할 필요 없음 — 재배포·CI를 1회로).

5. **배치 계속? / flush 확인** — 커밋 해시를 사용자에게 보여주고
   **"더 처리할 이슈가 있나요? 없으면 모아둔 커밋을 한 번에 push하겠습니다"** 라고 묻는다.
   - **더 있으면:** 이 커밋을 배치에 남겨둔 채 다음 이슈로(6단계로 가지 않음). 이슈는 **열린 채** 유지.
   - **지금 push하면:** 6단계(flush)로.

6. **flush — push 후 일괄 마감** — 사용자가 push를 지시하면:
   1. **(선택 — 문서 거버넌스가 있는 프로젝트)** 진행 문서(CHANGELOG·progress 문서 등)를 쓰는 프로젝트라면
      이번 배치에서 처리한 이슈들의 기록을 **한 커밋(`docs: ...`)으로 모아** 반영한다(이슈별로 쪼개지 않음).
   2. `git push origin <현재브랜치>` **한 번**으로 미push 커밋 전체를 민다. (force·다른 브랜치·다른 remote 금지 — 훅/권한이 차단.)
   3. **push 성공을 확인한 뒤에야**, 이번 배치에서 resolve된 **각 이슈**(= 방금 push된 `(#N)` 커밋들 중 아직 열려 있고 [템플릿3] 없는 것)에
      **[템플릿 3]** 코멘트를 `--body-file` 로 게시하고 `gh issue close <N>` 한다.
   > 🔒 **불변식:** [템플릿3]·close 는 **반드시 push 성공 후.** 아직 원격에 없는 커밋을 "push 완료"로 close하지 않는다.
   > push 가 실패하면 커밋은 로컬에 남기고 이슈는 열어둔 채 멈춘다(로그에 "push 실패/보류").

   **[템플릿 3]**
   ```markdown
   ## ✅ 결정에 따라 처리되었습니다

   위 코멘트에서 제시된 옵션 중 **옵션 <X>**로 결정되어 구현되었습니다.

   - **선택 이유:** <담당자가 밝힌 이유 또는 대화 요약>
   - **커밋:** `<커밋 해시>`
   - **처리 시각:** <YYYY-MM-DD HH:MM>
   ```

7. **로그** — 마감한 각 이슈마다 `.claude/automation/log.md` 에 append:
   `- [이슈] #<N>: 옵션 <옵션> 선택 → 구현 → 커밋 <해시> → (배치) push 완료 → close 완료`.

> **배치 대기(커밋했으나 아직 push 전)** 인 이슈는 열린 상태로 남는다. 세션이 끊겨도
> [`/session-briefing`](session-briefing.md) 의 "미push 커밋" 단계가 이를 잡아 재개(push → [템플릿3] → close)한다.
